# Генерация пар букв из словаря

**Дата:** 2026-07-18
**Автор:** anaalexovschik
**Статус:** одобрен, готов к плану реализации

## Контекст

Словарич — TWA-игра для двух игроков. Каждый раунд выдаёт две случайные буквы
кириллицы (первая и последняя); игроки устно подбирают слово, начинающееся на
первую и заканчивающееся на последнюю.

Текущий генератор `src/lib/letters.ts:15` выбирает первую и последнюю буквы
независимо из алфавита за вычетом нескольких очевидных плохих букв
(`Ъ Ь Ы Й` в начале, `Ъ Ь` в конце). Никакой проверки, что для выбранной пары
реально существует русское слово, нет. Поэтому в игре могут выпадать пары вида
`Ц..Э`, `Ъ..Я` (теоретически), и подобные — на которые у игроков нет шанса
придумать слово, и раунд превращается в тык.

## Цель

Генератор должен выдавать только те пары `(first, last)`, для которых в словаре
существует хотя бы несколько реальных русских слов. Дополнительно — показать
примеры слов по запросу игрока (кнопка «Не помню слово?»), чтобы пара не
становилась тупиком.

## Не-цели

- Полная валидация названного слова через input — игроки по-прежнему проверяют
  слова устно.
- Хранение полного словаря в бандле приложения — нужен только список валидных
  пар с примерами.
- Сложные режимы сложности — на первом шаге один порог для всех.

## Архитектура

Build-time прекомпьют. Словарь живёт в репозитории как текстовые файлы.
Node-скрипт читает их, группирует слова по `(first, last)`, отбраковывает
редкие пары и пишет JSON, который импортируется в приложение на этапе сборки.

### Файловая раскладка

```
data/
├── raw-nouns.txt              сторонний список лемм, ~36 000 строк
└── extra-words.txt            ручной сленг / имена собственные

scripts/
└── build-pairs.mjs            build-time скрипт: raw → letterPairs.json

src/data/
└── letterPairs.json           генерируется, коммитится в git

src/lib/
└── letters.ts                 переписывается: сэмплит из letterPairs.json

src/lib/__tests__/
└── letters.test.ts            УДАЛЯЕТСЯ
```

### Поток данных

1. **Разовый/редкий шаг** (вручную, раз в несколько месяцев): обновить
   `data/raw-nouns.txt` из upstream.
2. **Разработка/CI:** `npm run build:pairs` перегенерирует
   `src/data/letterPairs.json`.
3. **Vite-сборка:** JSON импортируется как модуль, попадает в бандл.
4. **Runtime:** `generateLetterPair(prev)` сэмплит случайную пару из JSON,
   избегая `prev`.

## Источник данных

### `data/raw-nouns.txt`

- Источник: <https://github.com/Harrix/Russian-Nouns>
- Лицензия: MIT
- Формат: plain text, одно слово на строку, UTF-8, нижний регистр
- Обновление: ручное. Скачать `russian_nouns.txt` из upstream, заменить локальный
  файл, закоммитить. Никаких авто-скачиваний в CI.

### `data/extra-words.txt`

Ручная кураторская добавка того, чего нет в литературном словаре.

- **Сленг / современные заимствования:** `кринж, вайб, краш, чилить, дроп,
  свайп, лайк, репост, стрим, флекс, база, имба, рофл, факап, дедлайн, митинг,
  стендап, ревью, деплой`
- **Имена собственные:** топонимы, бренды, имена — `Москва, Камчатка, Царицыно,
  Байкал, Гугл, Тинькофф, Яндекс`

Формат:
- Одно слово на строку
- Строки, начинающиеся с `#` — комментарии, игнорируются скриптом
- Регистр произвольный (скрипм нормализует через `toLowerCase()`)

Принцип пополнения: если игроки жалуются, что какая-то пара
появляется слишком часто / слишком сложная / наоборот, в `extra-words.txt`
добавляются слова, после чего `npm run build:pairs` пересобирает JSON.

## Поведение build-скрипта

`scripts/build-pairs.mjs`, Node ESM. Шаги по порядку:

1. **Чтение.** Загрузить `data/raw-nouns.txt` и `data/extra-words.txt` в один
   массив. Пропустить пустые строки и строки, начинающиеся с `#`.
2. **Нормализация каждого слова:**
   - `toLowerCase()` + `trim()`
   - `Ё` → `Е` (также `ё` → `е`)
   - Отбраковать слово целиком, если содержит любой символ вне русского
     алфавита `А-Яа-я` (после нормализации Ё). Конкретно: дефис, пробел, цифры,
     латиница, точки → слово в мусор.
3. **Фильтр по длине:** `word.length >= 3`.
4. **Дедупликация:** через `Set`.
5. **Группировка:** по ключу `(first, last)` — обе буквы в верхнем регистре.
6. **Порог:** оставить только пары с `count >= 3`.
7. **Выбор примеров для каждой пары:** топ-3 самых коротких слова. При равенстве
   длины — по алфавиту.
8. **Сортировка результата:** по `first`, затем по `last`. Детерминированный
   вывод, чтобы diff-ы в git были минимальными.
9. **Защита от регрессии:** если после всех фильтров осталось меньше 50 пар —
   скрипт падает с ошибкой. Страховка от того, что кто-то испортил
   `raw-nouns.txt` или изменил параметры и сломал пул пар.
10. **Запись** `src/data/letterPairs.json`.

## Формат `src/data/letterPairs.json`

```json
{
  "version": 1,
  "generatedAt": "2026-07-18T10:30:00Z",
  "threshold": { "minWords": 3, "minLength": 3 },
  "pairs": [
    { "first": "А", "last": "А", "count": 9, "examples": ["ара", "абака", "абразия"] },
    { "first": "А", "last": "К", "count": 47, "examples": ["аул", "арка", "азбук"] }
  ]
}
```

- `version` — для будущих миграций формата. Сейчас `1`.
- `generatedAt` — ISO timestamp момента запуска скрипта.
- `threshold` — зафиксированные параметры генерации (для отладки).
- `pairs[]` — отсортированный массив. Каждая пара уникальна по `(first, last)`.

Размер: ~15–25 КБ сырого JSON, ~5–8 КБ после gzip.

## Изменения в коде приложения

### `src/lib/letters.ts` (переписывается)

```ts
import type { LetterPair } from '../types'
import pairData from '../data/letterPairs.json'

export interface LetterPairWithExamples extends LetterPair {
  count: number
  examples: string[]
}

const PAIRS: LetterPairWithExamples[] = pairData.pairs
const FALLBACK: LetterPair = { first: 'А', last: 'К' }

export function generateLetterPair(prev?: LetterPair): LetterPair {
  if (PAIRS.length === 0) return FALLBACK
  for (let i = 0; i < 50; i++) {
    const candidate = PAIRS[Math.floor(Math.random() * PAIRS.length)]
    if (!prev) return candidate
    if (candidate.first !== prev.first || candidate.last !== prev.last) {
      return candidate
    }
  }
  return FALLBACK
}

export function getExamples(pair: LetterPair): string[] {
  const found = PAIRS.find(p => p.first === pair.first && p.last === pair.last)
  return found?.examples ?? []
}
```

## Открытые вопросы для этапа реализации

1. **Размер `raw-nouns.txt` в git.** ~500 КБ текста. Сжимается хорошо. На первом
   шаге просто коммитим как есть; git LFS — если позже станет проблемой.
2. **Что входит в стартовый `extra-words.txt`.** Минимальный набор — ~30–50
   слов из приведённого выше списка, расширяется по мере жалоб от игроков.
3. **Финальная визуальная трактовка кнопки «Не помню слово?»** — текст линком
   или ghost-кнопкой, точные отступы. Решается на этапе UI-реализации.
