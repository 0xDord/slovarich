Используя свой инструмент (навык) superpowers, полностью сгенерируй, настрой и подготовь к запуску локальный проект Telegram Web App (TWA). Это игра в слова для двух игроков на одном устройстве.

Технологический стек:

React + TypeScript + Vite.

Tailwind CSS для стилизации.

@twa-dev/sdk или @telegram-apps/sdk для интеграции с Telegram.

Требования к UI/UX (Telegram Style):

Дизайн должен строго следовать нативному стилю Telegram.

Используй системные CSS-переменные Telegram (например, var(--tg-theme-bg-color), var(--tg-theme-text-color), var(--tg-theme-button-color)) в конфигурации Tailwind или напрямую в стилях, чтобы приложение бесшовно адаптировалось под светлую/темную тему мессенджера.

Отключи стандартное выделение текста и скролл (приложение должно ощущаться как нативная полноэкранная игра).

Механика игры (Local Mode):

Генерация задания: При старте раунда алгоритм выдает две случайные буквы (первую и последнюю), между которыми скрыты остальные буквы (например, "А ... К").

Интерфейс: Экран разделен по горизонтали на две равные тач-зоны (Player 1 сверху, перевернут на 180 градусов для удобства; Player 2 снизу). В центре экрана отображаются заданные буквы.

Buzzer-механика: Игрок, который первым вспомнил подходящее слово, нажимает на свою половину экрана.

Ввод и проверка: После нажатия игра ставится на паузу. У нажавшего игрока появляется поле ввода (Input). Он должен напечатать придуманное слово.

Валидация: Напиши простую функцию-заглушку validateWord(word, firstLetter, lastLetter), которая проверяет, что слово действительно начинается и заканчивается на заданные буквы (в будущем мы подключим туда реальный словарь).

Подсчет очков: Если слово валидно, игрок получает +1 балл. Если нет — штраф или переход хода. Общий счет постоянно отображается на экране.

Инструкция для выполнения (Superpowers):

Создай директорию проекта, инициализируй React+Vite приложение.

Установи все необходимые npm-пакеты (Tailwind, TWA SDK, Lucide React для иконок).

Настрой tailwind.config.js для работы с переменными Telegram.

Создай компонент игрового поля с логикой состояний (ожидание, нажатие баззера, ввод слова, обновление счета).

Напиши README.md с командой для запуска локального сервера.
